## <a name="azure-cli-20"></a>Azure CLI 2.0
이러한 단계를 수행하려면 Azure CLI 버전 2.0.4 이상이 필요합니다. `az --version`을 실행하여 버전을 찾습니다. 업그레이드해야 하는 경우 [Azure CLI 2.0 설치]( /cli/azure/install-azure-cli)를 참조하세요.

먼저 [az group create](/cli/azure/group#create)를 사용하여 리소스 그룹을 만듭니다. 그런 다음 [az group deployment create](/cli/azure/group/deployment#create)를 사용하여 템플릿에서 배포를 만듭니다. 다음과 같이 `--template-uri` 매개 변수와 `--parameters`에 필요한 모든 템플릿 매개 변수를 사용하여 전체 URI를 템플릿에 제공합니다.

```azurecli
az group create --name myResourceGroup --location eastus
az group deployment create --resource-group myResourceGroup \
    --template-uri https://raw.githubusercontent.com/azure/azure-quickstart-templates/master/{template-name}/azuredeploy.json \
    --parameters '{"sshKeyData": {"value": "ssh-rsa AAAAB3N{snip}B9eIgoZ"}}'
```

템플릿에 따라 Azure의 리소스 배포에 다소 시간이 걸릴 수 있습니다.


## <a name="azure-powershell"></a>Azure PowerShell
이러한 단계를 수행하려면 Azure PowerShell 모듈 버전 3.6 이상이 필요합니다. `Get-Module -ListAvailable AzureRM`을 실행하여 버전을 찾습니다. 업그레이드해야 하는 경우 [Azure PowerShell 모듈 설치](/powershell/azure/install-azurerm-ps)를 참조하세요.

먼저 [New-AzureRmResourceGroup](/powershell/module/azurerm.resources/new-azurermresourcegroup)을 사용하여 리소스 그룹을 만듭니다. 그런 다음 [New-AzureRmResourceGroupDeployment](/powershell/module/azurerm.resources/new-azurermresourcegroupdeployment)를 사용하여 템플릿에서 배포를 만듭니다. 다음과 같이 `-TemplateURI` 매개 변수를 사용하여 전체 URI를 템플릿에 제공합니다.

```powershell
New-AzureRmResourceGroup -Name "myResourceGroup" -Location "EastUS"
New-AzureRmResourceGroupDeployment -ResourceGroupName "myResourceGroup" `
     -TemplateUri "https://raw.githubusercontent.com/azure/azure-quickstart-templates/master/{template-name}/azuredeploy.json"
```

[New-AzureRmResourceGroupDeployment](/powershell/module/azurerm.resources/new-azurermresourcegroupdeployment) 명령을 실행하면 템플릿의 매개 변수에 대한 값을 입력하라는 메시지가 표시될 수 있습니다. 템플릿에 따라 Azure의 리소스 배포에 다소 시간이 걸릴 수 있습니다.
