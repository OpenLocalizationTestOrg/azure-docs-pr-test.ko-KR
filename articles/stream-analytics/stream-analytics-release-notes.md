| 메모리 부족 문제 |된 순서가의 이벤트 및/또는 복잡 한 쿼리 작업에 메모리가 부족 hello 작업 toorun 않을 수 많은 양의 상태를 유지 관리에 대 한 큰 허용 스트리밍 분석 작업을 다시 시작 합니다. hello 시작 및 중지 작업 hello 작업의 작업 로그에 표시 됩니다. tooavoid이 동작, 여러 파티션에서 축소 배율 hello 쿼리 합니다. 향후 릴리스에서는 영향 받는 작업을 다시 시작하지 않고 성능을 저하시켜 이러한 제한 문제를 해결할 예정입니다. |
| 페이로드 타임스탬프가 없는 대용량 Blob 입력은 메모리 부족 문제를 일으킬 수 있습니다. |Blob 저장소에서 큰 파일을 사용 하 여 타임 스탬프를 통해 타임 스탬프 필드를 지정 하지 않은 경우 스트림 분석 작업 toocrash를 발생할 수 있습니다. tooavoid이 문제를 크기가 10MB 미만 각 blob을 유지 합니다. |
| SQL 데이터베이스 이벤트 볼륨 제한 |출력 대상으로 SQL 데이터베이스를 사용할 때 매우 많은 양의 출력 데이터 아웃 hello 스트림 분석 작업 tootime를 발생할 수 있습니다. 이 문제를 하거나 tooresolve 집계 또는 필터 연산자를 사용 하 여 hello 출력 볼륨을 줄이거나 출력 대상으로 Azure Blob 저장소 또는 이벤트 허브를 대신 선택 합니다. |
| PowerBI 데이터 집합은 하나의 테이블만 포함할 수 있습니다. |PowerBI는 지정된 데이터 집합에서 둘 이상의 테이블을 지원하지 않습니다. |
