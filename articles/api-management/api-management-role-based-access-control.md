## <a name="custom-roles"></a><span data-ttu-id="96949-147">사용자 지정 역할</span><span class="sxs-lookup"><span data-stu-id="96949-147">Custom Roles</span></span>

<span data-ttu-id="96949-151">사용자 지정 역할을 만들 때는 기본 제공 역할 중 하나로 시작하는 것이 쉽습니다.</span><span class="sxs-lookup"><span data-stu-id="96949-151">When you create a custom role, it is easier to start with one of the built-in roles.</span></span> <span data-ttu-id="96949-152">속성을 편집하여 Actions, NotActions 또는 AssignableScopes를 추가한 다음 변경 사항을 새 역할로 저장합니다.</span><span class="sxs-lookup"><span data-stu-id="96949-152">Edit the attributes to add the Actions, NotActions, or AssignableScopes, then save the changes as a new role.</span></span> <span data-ttu-id="96949-153">다음 예제는 "Azure API Management 서비스 독자" 역할로 시작하고 "Calculator API Editor"라는 사용자 지정 역할을 만듭니다.</span><span class="sxs-lookup"><span data-stu-id="96949-153">The following example begins with the "Azure API Managment Service Reader" role and creates a custom role called "Calculator API Editor".</span></span> <span data-ttu-id="96949-154">사용자 지정 역할은 특정 API에만 할당될 수 있으므로 해당 API에 대한 액세스 권한만 보유합니다.</span><span class="sxs-lookup"><span data-stu-id="96949-154">The custom role can be assigned only to a specific API therefore will only has access to that API.</span></span> 

```
$role = Get-AzureRmRoleDefinition "API Management Service Reader Role"
$role.Id = $null
$role.Name = 'Calculator API Contributor'
$role.Description = 'Has read access to Contoso APIM instance and write access to the Calculator API.'
$role.Actions.Add('Microsoft.ApiManagement/service/apis/write')
$role.AssignableScopes.Clear()
$role.AssignableScopes.Add('/subscriptions/<subscription ID>/resourceGroups/<resource group name>/providers/Microsoft.ApiManagement/service/<service name>/apis/<api ID>')
New-AzureRmRoleDefinition -Role $role
New-AzureRmRoleAssignment -ObjectId <object ID of the user account> -RoleDefinitionName 'Calculator API Contributor' -Scope '/subscriptions/<subscription ID>/resourceGroups/<resource group name>/providers/Microsoft.ApiManagement/service/<service name>/apis/<api ID>'
```
