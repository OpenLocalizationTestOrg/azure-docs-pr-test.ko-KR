# <a name="how-to-use-the-azure-marketplace-seller-insights-portal"></a>Azure Marketplace Seller Insights 포털을 사용하는 방법

이 문서에서는 관리자가 Seller Insights 포털에 액세스하고, 사용자를 추가 또는 삭제하고, 사용자에 대해 사용 권한을 설정하는 방법을 설명합니다.

## <a name="onboard-users"></a>사용자 등록

이 섹션에서는 관리자가 Seller Insights 포털에 처음 액세스하는 방법을 설명합니다. 이 섹션의 작업은 관리자가 처리해야 합니다. 


### <a name="overview-of-roles"></a>역할 개요

Seller Insights 포털 관리는 사용자를 추가, 편집 및 삭제합니다.

Seller Insights 포털 사용자는 조직에서 사용할 수 있는 모든 데이터를 볼 수 있습니다.

관리자와 사용자는 유효한 [Microsoft 계정](http://signup.live.com) 또는 조직의 회사 또는 학교 계정을 사용하여 인증해야 합니다. 새 계정 ID가 모니터링된 사서함과 연결되어 있는지 확인합니다.

![계정 유형][5]



### <a name="invitation-to-activate-your-account"></a>계정 활성화를 위한 초대

관리자는 Seller Insights 포털에 로그인하기 전에 사용자로 설정하려는 개인의 계정을 식별해야 합니다. 회사 또는 학교 계정이나 Microsoft 계정으로 사용자를 설정합니다.

서비스를 활성화하려면 관리자는 초대 전자 메일에 표시되는 [URL 주소](https://reports.azure.com)로 이동해야 합니다.

>[!NOTE]
>관리자 전자 메일이 Microsoft 계정으로 나열되어 있고 초대를 받은 전자 메일 주소와 연결된 Microsoft 계정을 만들지 않은 경우 로그인하기 전에 [계정을 만들어야](https://signup.live.com) 합니다.


### <a name="administrator-sign-in-to-the-seller-insights-portal"></a>Seller Insights 포털에 대한 관리자 로그인

1. 관리자에게 보낸 전자 메일의 URL을 클릭하여 Seller Insights 포털에 로그인합니다.
2. 관리자 전자 메일 주소와 암호를 입력합니다.
3. **로그인** 단추를 클릭합니다.
4. 계정 관리를 시작합니다. 사용자를 추가하고 권한을 설정하고 사용자를 편집합니다.
5. 사용자에게 Seller Insights 포털에 대한 URL이 있는지 확인합니다.

    ![로그인 페이지][6]

## <a name="user-sign-in-to-the-portal"></a>포털에 대한 사용자 로그인

관리자는 계정이 만들어졌다는 사실을 전자 메일을 통해 사용자에게 알리고 Seller Insights 포털의 [URL 주소](https://reports.azure.com)를 사용자와 공유해야 합니다.

>[!NOTE]
>관리자는 Azure Marketplace에 솔루션을 게시하는 계정의 소유자입니다. 이 계정의 소유자는 다른 사용자에게 액세스 권한을 제공할 수 있습니다.  게시 포털에 공동 관리자 권한으로 액세스하는 경우 다음 단계를 수행하여 관리자 주소를 찾을 수 있습니다.
>1. 게시 포털에 로그인합니다.
>2. **게시자** 탭을 클릭합니다.
>3. 문제 게시자(일반적으로 하나만)를 엽니다.
>4. 왼쪽 아래에서 **관리자**를 클릭합니다.
>5. 찾으려는 관리자 전자 메일 주소는 레이블이 *소유자*로 지정되어 있습니다.


개인 Microsoft 계정이나 회사 또는 학교 계정을 사용하여 Seller Insights 포털에 로그인할 수 있습니다. 계정 유형은 프로필의 관리자 설정에 따라 달라집니다. 관리자는 사용할 계정을 사용자에게 알려야 합니다.

![계정 유형][1]



**Seller Insights 포털을 시작합니다.**

1. 로그인하면 **요약** 페이지로 이동됩니다.
2. 잠시 시간을 내어 개인 정보 취급 방침을 검토하세요.
3. 언제든지 Seller Insights 포털에 관련된 의문 사항이 포함된 메일을 통해 **문의**할 수 있습니다.
4. 페이지 맨 위에 있는 탭을 사용하여 Seller Insights 포털에서 이동할 수 있습니다.  Seller Insights 포털에서 현재 위치를 확인하려면 이 가이드의 페이지 맨 위에 있는 탭을 찾아 보세요.
5. 언제든지 스마일 아이콘을 클릭하여 Seller Insights 포털에 관련된 피드백을 보냅니다.
6. Seller Insights 포털에서 로그아웃하려면 오른쪽 위 구석에 있는 자신의 이름을 클릭합니다.

    ![기본 방문 페이지][2]


## <a name="summary-view"></a>요약 보기


1. **요약** 탭을 선택합니다.
2. 여기서 관심 있는 기간의 **시작 날짜**를 입력한 다음 **적용**을 클릭합니다. 3개월 동안의 데이터가 표시됩니다.
3. 선택한 3개월 동안의 새 주문 추세 데이터가 여기에 표시됩니다.
4. 선택한 기간의 새 주문은 채널 및 제안 형식으로 여기에 표시됩니다.
5. 이 패널에서 보기를 선택합니다. **상위 5대 시장**, **상위 5대 제안 이행** 또는 **상위 5대 고객**.
6. 유료 제안에 대한 선택한 기간의 지급 추세가 여기에 표시됩니다. BYOL 또는 자사 제안만 있는 경우에는 이 패널이 표시되지 않습니다.

    ![요약 보기][7]

## <a name="orders-and-usage"></a>주문 및 사용 현황
### <a name="overview"></a>개요

*패널 A*에서 **주문 및 사용 현황** 탭을 선택할 수 있습니다.  **보고서 종류**에서 **주문** 또는 **사용 현황**을 선택한 다음 슬라이더를 사용하여 관심 있는 데이터를 선택할 수 있습니다.

*패널 B*에서 월별 데이터를 선택할 수 있습니다. 패널 A에서 **보고서 종류**의 선택에 따라 **주문** 또는 **사용 현황**을 선택합니다. 표시되는 데이터는 입력한 날짜 및 패널 A에서 슬라이더를 사용하여 적용한 필터를 반영합니다.

*패널 C*에서 3개월 추세 데이터를 선택할 수 있습니다. 패널 A에서 **보고서 종류**의 선택에 따라 **주문** 또는 **사용 현황**을 선택합니다. 표시되는 데이터는 입력한 날짜 및 패널 A에서 슬라이더를 사용하여 적용한 필터를 반영합니다.

*패널 D*에서 각 개인에 대한 구독 데이터가 나열됩니다.  이 패널에서 데이터를 Excel 또는 CSV 형식으로 다운로드할 수 있습니다.

![주문 개요][8]
![주문 개요 맵][9]


### <a name="orders"></a>주문
#### <a name="monthly-view-panel-a"></a>월별 보기: 패널 A

![패널 맵 A][10]

  1. 주문 및 사용 현황 데이터에 액세스하려면 위쪽 행의 **주문 및 사용 현황** 탭을 선택합니다.
  2. **보고서 종류**에서 **주문**을 선택합니다.
  3. 필터 기호를 사용하여 슬라이서 옵션을 표시하거나 숨깁니다.  이러한 슬라이서를 사용하여 **Azure 라이선스 유형**, **Marketplace 라이선스 유형**, **시장**, **제안 형식** 및 **미리 보기** 상태별로 보려는 데이터를 필터링합니다.  이러한 용어에 대한 정의는 **도움말** 탭을 참조하세요.
  4. 관심 있는 시작 날짜를 선택한 다음 **이동**을 클릭합니다.

  ![패널 A 월별 주문 보기][14]

#### <a name="monthly-view-panel-b"></a>월별 보기: 패널 B

  ![패널 맵 B][11]

  1. **월별 보기**를 선택합니다.
  2. 주문 요약 정보는 채널별로 표시됩니다(CSP, Direct Enterprise, Indirect Enterprise 및 Web Direct).
  3. **월별 보기**의 경우 **주문 요약 정보** 패널에서는 다음과 같은 차트 보기를 제공합니다.

* 시장별 주문(상위 10개)
* 제안 제목별 주문(상위 10개)
* Azure 라이선스 유형별 주문
* Marketplace 라이선스 유형별 주문


![패널 B 월별 주문 보기][15]

#### <a name="trend-view-panel-c"></a>추세 보기: 패널 C

  ![패널 맵 C][12]

  1. **추세 보기**를 선택합니다.
  2. **주제 영역** 목록에서 보려는 항목을 선택한 다음 **적용**을 클릭합니다. 기본값은 **모두 선택**입니다.

  ![패널 C 월별 주문 보기 주제 영역 목록][17]

  3. 필터 기호를 사용하여 모든 선택 항목에 대해 **모두**(모든 데이터) 또는 **상위 10**을 선택합니다. 기본값은 **상위 10**입니다.

  ![패널 C 월별 주문 보기 필터 기호][18]

  4. 선택한 차트의 각 막대를 가리켜 기본 데이터를 표시합니다.
  5. **+** 기호를 사용하여 선택한 각 차트의 아래에 데이터가 나타나는 모드를 전환합니다.

  ![패널 C 주문 추세 보기][16]



#### <a name="detailed-orders-data-panel-d"></a>자세한 주문 데이터: 패널 D

  ![패널 맵 D][13]

  1. 주문 세부 정보 패널에는 자세한 **주문 데이터**가 표시됩니다.
  2. 각 열의 맨 위에 있는 필터를 사용하여 다양한 명령과 값으로 데이터를 필터링합니다.

  ![패널 D 월별 주문 보기 필터][20]

  3. 보려는 페이지당 항목 수를 선택합니다. 페이지당 10, 20 또는 50 항목을 사용할 수 있습니다.
  4. 오프라인 분석을 위해 데이터를 Excel 또는 CSV 형식으로 다운로드합니다.

  ![패널 D 자세한 주문 보기][19]

### <a name="accessing-detailed-customer-data"></a>자세한 고객 데이터 액세스

  1. **주문 세부 정보** 패널에서 관심 있는 Azure 구독 ID를 클릭합니다.  비BYOL Marketplace 라이선스 유형 ID만 클릭할 수 있습니다. BYOL 구독 ID는 보안상의 이유로 해시됩니다.

  ![주문 및 사용 현황의 고객 세부 정보 패널][26]

  2. **고객 세부 정보 데이터** 패널로 리디렉션됩니다. 고객에 대한 데이터가 검색됩니다.
  3. 데이터를 Excel 또는 CSV 형식으로 다운로드할 수 있습니다.

  ![주문 및 사용 현황의 고객 세부 정보 패널][27]

### <a name="customer-data"></a>고객 데이터

  1. **고객** 탭을 선택합니다.
  2. 슬라이서를 사용하여 **Azure 라이선스 유형** 또는 **Marketplace 라이선스 유형**별로 보려는 데이터를 필터링합니다.
  3. 관심 있는 기간의 **시작 날짜** 및 **종료 날짜**를 선택합니다.
  4. **Marketplace 라이선스 유형**별 고객 요약 정보가 표시됩니다.
  5. 각 열의 맨 위에 있는 필터를 사용하여 다양한 명령과 값으로 데이터를 필터링합니다.
  6. 데이터를 Excel 또는 CSV 형식으로 다운로드합니다.

  ![고객 데이터 패널][28]


## <a name="user-management"></a>사용자 관리

이 섹션의 작업은 관리자가 처리해야 합니다. 다음 절차를 사용하여 Seller Insights 포털에서 사용자를 추가하거나, 사용 권한을 변경하거나, 사용자를 삭제합니다.


### <a name="add-user"></a>사용자 추가

이 섹션에서는 Seller Insights 포털에서 새 사용자를 추가하거나 해당 권한을 변경하는 방법을 보여 줍니다.


1. **사용자 관리** 탭을 선택합니다.
2. 사용자를 추가하려면 선택 **사용자 추가**를 선택합니다.
3. 해당 Microsoft 계정이나 회사 또는 학교 계정과 연결된 전자 메일 주소를 입력합니다. 그런 후 **계속**을 클릭합니다.
4. 관리자는 사용자에게 읽기 전용 보고서 액세스 권한을 할당할 수 있습니다.
5. 사용자 세부 정보(**이름**, **성** 및 **전자 메일**)를 입력한 다음 **계속**을 클릭합니다.
6. 사용자에게 할당하려는 권한을 선택한 다음 **계속**을 클릭합니다.
7. 입력한 정보가 정확한지 확인한 다음 **저장**을 클릭하여 계속합니다.  사용자를 성공적으로 추가했습니다.

  ![사용자 추가 패널][29]


### <a name="edit-user"></a>사용자 편집

1. **사용자 관리** 탭을 선택한 다음 **사용자 편집**을 선택합니다.
2. 수정하려는 권한을 가진 사용자를 찾습니다.  새 사용자를 추가할 수도 있습니다. **사용자 추가**를 클릭합니다.
3. 해당 사용자에 대해 제거하려는 권한을 클릭합니다.  사용자를 삭제할 수도 있습니다. **사용자 삭제**를 클릭합니다.
4. **저장**을 클릭하여 계속합니다.  사용자의 권한을 성공적으로 편집했습니다.

  ![사용자 편집 패널][30]

## <a name="help-support-requests"></a>도움말 지원 요청


### <a name="access-the-support-request-format"></a>지원 요청 서식 액세스

1. **도움말** 탭을 선택합니다.
2. **지원 및 문서**를 선택한 다음 **요청 시작**을 선택합니다.
3. **문의처** 링크에서 지원 요청을 제출할 수도 있습니다. 이 링크는 응용 프로그램의 각 페이지 맨 아래에 나타납니다.
4. 지원 요청을 제출할 수 있는 위치에서 새 탭이 열립니다.

  ![지원 양식 액세스][31]

### <a name="review-helpful-information"></a>유용한 정보 검토

1. 보고서와 관련된 지원 요청을 제출하려면 **문제 유형**에서 **보고서 및 정보**를 선택합니다.
2. 범주를 선택합니다.  옵션은 **보고서와 관련된 질문** 및 **기타**입니다.
3. 쿼리에 대한 답변을 제공할 수 있는 몇 가지 유용한 링크가 표시됩니다.  지원 요청을 제출하기 전에 이러한 링크를 확인합니다.
4. 그래도 지원 요청을 제출해야 하는 경우 **요청 시작**을 선택합니다.

  ![사용자 편집 패널][32]


### <a name="enter-information-and-submit-the-request"></a>정보 입력 및 요청 제출

1. 연락처 정보, 표준 시간대, 지역 및 언어 기본 설정을 입력한 다음 **계속**을 선택합니다.
2. 지원 요청에 대한 제목을 지정하고 최대한 많은 정보를 제공합니다.  또한 스크린샷과 같은 지원 파일을 업로드한 다음 **제출**을 선택할 수도 있습니다.
3. 지원 요청이 성공했음을 확인하는 메시지가 표시됩니다.
4. 언제든지 지원 요청 상태를 확인할 수 있습니다.

  ![지원 양식 액세스][33]


## <a name="reset-a-password"></a>암호 재설정

**계정 및 암호 유효성 검사**

1. *사용 중인 계정이 등록되어 있고 암호가 작동하지 않는지 확인합니다.* Azure Active Directory에 등록되어 있는 계정을 입력하면 계정으로 로그인하는 방법을 선택하라는 메시지를 자동으로 표시합니다. 개인 계정으로 사용되는 회사 또는 학교 계정이나 Microsoft 계정을 사용합니다.
2. [https://reports.azure.com](https://reports.azure.com)으로 이동하고 로그인될 수 있는 모든 계정을 로그아웃합니다. 다른 계정이 로그인할 수 없도록 개인 창 내에서 열 수도 있습니다.
3. 암호를 입력하지 않고 게시자 메일 주소를 입력합니다. Azure Active Directory에 등록되어 있는 경우 4단계로 리디렉션됩니다.

    >[!NOTE]
    >관리자 계정이 등록됩니다. 관리자 계정으로 설정되는 회사 또는 학교 계정도 등록됩니다.

4. **회사 또는 학교 계정**을 선택합니다. 회사 또는 학교 계정은 Azure에 등록된 회사에서 설정됩니다. 

    >[!NOTE]
    >사용자는 조직의 전자 메일 주소를 사용하여 Microsoft 계정을 설정할 수 있습니다. 이 시나리오에서 조직은 나중에 Azure로 등록되고 동일한 전자 메일 주소가 회사 또는 학교 계정으로 등록됩니다. 이 사용자는 회사 또는 학교 계정을 만드는 데 사용한 암호와 같은 암호를 갖는 Microsoft 계정을 유지할 수 있습니다. 이 사용자는 조직에서 사용되는 별도 암호로 회사 또는 학교 계정을 유지할 수도 있습니다.

5. Seller Insights 포털의 경우 **회사 또는 학교 계정**을 선택할 때 Microsoft 계정 암호가 아닌 회사 또는 학교 계정 암호를 사용하는지 확인합니다.
6. 회사 또는 학교 계정을 사용하면 로그인할 수 있게 게시자의 페이지로 리디렉션됩니다.
7. 여기에서 회사 또는 학교 계정의 암호가 작동하는지 확인합니다. 작동하지 않으면 이 문서의 "암호 다시 설정" 섹션으로 진행합니다.



**암호 재설정**

다음 Microsoft 및 회사 또는 학교 계정을 사용하여 암호를 재설정합니다.

>[!NOTE]
>관리자 계정이 설정한 회사 또는 학교 계정의 경우 관리자가 사용자가 직접 암호를 재설정할 수 있는 기능을 켠 경우에는 사용자가 암호를 재설정할 수 있습니다. 해당 기능을 켜지 않은 경우에는 관리자에게 문의할 수 있는 암호 재설정 양식이 제공됩니다.

* [Microsoft 계정](https://account.live.com/ResetPassword.aspx)
![Microsoft 계정 암호 재설정][3]

* [회사 또는 학교 계정](https://passwordreset.microsoftonline.com/)
![회사 또는 학교 계정 암호 재설정][4]
