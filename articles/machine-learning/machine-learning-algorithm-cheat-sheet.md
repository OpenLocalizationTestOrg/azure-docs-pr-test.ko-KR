  * <span data-ttu-id="12c94-142">**보충 학습**에서 알고리즘은 각 데이터 요소에 대한 응답으로 작업을 선택합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-142">In **reinforcement learning**, the algorithm gets to choose an action in response to each data point.</span></span> <span data-ttu-id="12c94-143">이는 지정 시간 센서 판독값 집합이 한 데이터 요소이며 알고리즘으로 로봇의 다음 동작을 선택해야 하는 로봇 공학의 일반적인 접근 방식입니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-143">It is a common approach in robotics, where the set of sensor readings at one point in time is a data point, and the algorithm must choose the robot’s next action.</span></span> <span data-ttu-id="12c94-144">사물 인터넷의 응용 프로그램에 적합한 학습이기도 합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-144">It's also a natural fit for Internet of Things applications.</span></span> <span data-ttu-id="12c94-145">또한 학습 알고리즘은 짧은 시간 후에 결정이 얼마나 효율적이었는지를 나타내는 보상 신호를 받습니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-145">The learning algorithm also receives a reward signal a short time later, indicating how good the decision was.</span></span> <span data-ttu-id="12c94-146">이 신호에 따라 알고리즘은 가장 높은 보상을 달성하기 위해 해당 전략을 수정합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-146">Based on this, the algorithm modifies its strategy in order to achieve the highest reward.</span></span> <span data-ttu-id="12c94-147">현재 Azure 기계 학습에는 보충 학습 알고리즘 모듈이 없습니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-147">Currently there are no reinforcement learning algorithm modules in Azure ML.</span></span>

* <span data-ttu-id="12c94-148">**베이지안 분석법**은 통계적으로 독립적인 데이터 요소를 가정합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-148">**Bayesian methods** make the assumption of statistically independent data points.</span></span> <span data-ttu-id="12c94-149">즉, 한 데이터 요소의 모델링되지 않은 가변성이 다른 데이터 요소와 상호 관련이 없어 예측할 수 없음을 의미합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-149">This means that the unmodeled variability in one data point is uncorrelated with others, that is, it can’t be predicted.</span></span> <span data-ttu-id="12c94-150">예를 들어 기록되는 데이터가 다음 지하철이 도착할 때까지 남은 시간(분)인 경우 하루 간격으로 수행된 두 측정값은 통계적으로 독립적입니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-150">For example, if the data being recorded is the number of minutes until the next subway train arrives, two measurements taken a day apart are statistically independent.</span></span> <span data-ttu-id="12c94-151">그러나 1분 간격으로 측정한 두 값은 통계적으로 독립적이지 않습니다. 한 측정값을 보면 다른 측정값을 쉽게 예측할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-151">However, two measurements taken a minute apart are not statistically independent - the value of one is highly predictive of the value of the other.</span></span>

* <span data-ttu-id="12c94-152">**승격된 의사 결정 트리 회귀 모델**은 기능 겹침 또는 기능 간의 상호 작용을 활용합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-152">**Boosted decision tree regression** takes advantage of feature overlap or interaction among features.</span></span> <span data-ttu-id="12c94-153">즉, 지정된 데이터 요소에서 한 기능의 값은 다른 기능의 값을 다소 예측합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-153">That means that, in any given data point, the value of one feature is somewhat predictive of the value of another.</span></span> <span data-ttu-id="12c94-154">예를 들어 일일 최고/최저 온도 데이터에서 최저 온도를 알면 최고 온도를 적절하게 추측할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-154">For example, in daily high/low temperature data, knowing the low temperature for the day allows you to make a reasonable guess at the high.</span></span> <span data-ttu-id="12c94-155">두 기능에 포함된 정보는 다소 중복됩니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-155">The information contained in the two features is somewhat redundant.</span></span>

* <span data-ttu-id="12c94-156">본질적으로 다중 클래스 분류자를 사용하거나 두 클래스 분류자 집합을 하나의 **앙상블**로 결합하면 데이터를 세 개 이상의 범주로 분류할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-156">Classifying data into more than two categories can be done by either using an inherently multi-class classifier, or by combining a set of two-class classifiers into an **ensemble**.</span></span> <span data-ttu-id="12c94-157">앙상블 접근 방식에서는 각 클래스에 대해 별도의 두 클래스 분류자가 있습니다. 각 분류자는 데이터를 두 개의 범주, 즉 "이 클래스" 및 "이 클래스 아님"으로 구분합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-157">In the ensemble approach, there is a separate two-class classifier for each class - each one separates the data into two categories:  “this class” and “not this class.”</span></span> <span data-ttu-id="12c94-158">그런 다음 이러한 분류자는 데이터 요소의 올바른 할당에 투표합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-158">Then these classifiers vote on the correct assignment of the data point.</span></span> <span data-ttu-id="12c94-159">이는 [One-vs-All Multiclass][one-vs-all-multiclass]의 작동 원리입니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-159">This is the operational principle behind [One-vs-All Multiclass][one-vs-all-multiclass].</span></span>

* <span data-ttu-id="12c94-160">로지스틱 회귀 및 Bayes 지점 컴퓨터를 포함한 몇 가지 메서드는 **선형 클래스 경계**를 가정합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-160">Several methods, including logistic regression and the Bayes point machine, assume **linear class boundaries**.</span></span> <span data-ttu-id="12c94-161">즉, 클래스 간의 경계가 대략 직선(또는 보다 일반적인 경우 초평면)이라고 가정합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-161">That is, they assume that the boundaries between classes are approximately straight lines (or hyperplanes in the more general case).</span></span> <span data-ttu-id="12c94-162">대체는 이는 구분하려고 시도할 때까지 알 수 없는 데이터 특성이지만, 일반적으로 미리 시각화하여 학습할 수 있는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-162">Often this is a characteristic of the data that you don’t know until after you’ve tried to separate it, but it’s something that typically can be learned by visualizing beforehand.</span></span> <span data-ttu-id="12c94-163">클래스 경계가 매우 불규칙한 경우 의사 결정 트리, 의사 결정 정글, 지원 벡터 컴퓨터 또는 신경망 방법을 유지합니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-163">If the class boundaries look very irregular, stick with decision trees, decision jungles, support vector machines, or neural networks.</span></span>

* <span data-ttu-id="12c94-164">각 범주에 대한 **더미 변수**를 만든 다음 범주가 적용되는 경우 1로, 적용되지 않는 경우 0으로 설정하여 신경망을 범주 변수와 함께 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="12c94-164">Neural networks can be used with categorical variables by creating a **dummy variable** for each category, setting it to 1 in cases where the category applies, 0 where it doesn’t.</span></span>
