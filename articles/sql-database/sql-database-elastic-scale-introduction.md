* <span data-ttu-id="092b8-157">테넌트는 물리적으로 서로 격리되어야 할 수 있으므로 각 테넌트에는 별도의 데이터베이스가 필요합니다.</span><span class="sxs-lookup"><span data-stu-id="092b8-157">Tenants may require physical isolation from each other, so separate databases are needed for each tenant</span></span>

<span data-ttu-id="092b8-162">분할은 응용 프로그램의 모든 트랜잭션이 분할 키의 단일 값으로 제한될 수 있는 경우에 가장 적합합니다.</span><span class="sxs-lookup"><span data-stu-id="092b8-162">Sharding works best when every transaction in an application can be restricted to a single value of a sharding key.</span></span> <span data-ttu-id="092b8-163">그렇게 하면 모든 트랜잭션이 특정 데이터베이스에 로컬이 됩니다.</span><span class="sxs-lookup"><span data-stu-id="092b8-163">That ensures that all transactions will be local to a specific database.</span></span>
